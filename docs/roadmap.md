# Roadmap

## Near Term

- improve table diagnosis rules from sample rows
- add more Excel and WPS compatibility guidance
- expand examples for lookup, cleanup, and summarization
- add common formula error repair patterns

This is the part that should make the repository more useful without making it much noisier.

## Mid Term

- support more multi-sheet reconciliation cases
- add report-oriented table preparation guidance
- document safer fallback workflows for older spreadsheet environments

This is where the repository should become more practical for messy real-world files, not just cleaner examples.

## Later

- add richer examples from real office data shapes
- refine category-specific guidance for sales, HR, finance, and operations tables

If things go well, later versions should feel less like notes and more like a dependable reference. Preferably without becoming boring in the wrong way.
